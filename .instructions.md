# VS Code Instructions for Records Explorer

This file provides instructions for working with the Records Explorer application in VS Code.

## Analyze Record Breaking Lifts

Generate a comprehensive report of lifts that would break existing records across all weight classes and age groups.

### Quick Start

```bash
npm run analyze-records
```

This command:
1. Fetches current records from the Google Sheets database
2. Analyzes top athletes for each weight class/age group combination
3. Identifies lifts that exceed existing records
4. Generates a markdown report: `record-breaking-analysis.md`

### What Gets Analyzed

**Weight Classes**: 18 total (9 women's, 9 men's) from 48kg to 109kg+

**Age Groups**: 11 categories including Open, Youth (U11-U17), Junior, Senior, and Masters (1-4)

**Total Combinations**: Up to 198 weight class/age group pairs

**Lift Types**: 
- Snatch (individual lift)
- Clean & Jerk (individual lift)
- Total (combined snatch + C&J)

### Output

The analysis generates `record-breaking-analysis.md` with three tables:

1. **Best Snatch** - Records sorted by snatch weight (descending)
2. **Best Clean & Jerk** - Records sorted by C&J weight (descending)
3. **Best Total** - Records sorted by total weight (descending)

Each table includes: Athlete name, weight, weight class, age group, current record, and lift date

### Requirements

- Internet connection (to fetch from APIs)
- Access to USAW API and Google Sheets (already configured)

### Performance

- Typical runtime: 5-15 minutes
- The script rate-limits requests to be respectful to the APIs
- All 198 weight class/age group combinations are checked

### Customization

Edit `scripts/analyze-records.js` to:
- Change how many top athletes to analyze (default: 5)
- Modify age groups or weight classes
- Adjust sorting or output format

See `SKILL.md` for detailed technical documentation.
